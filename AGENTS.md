# AGENTS.md ‚Äî my-hyde-dotfiles

## Role

Dotfiles maintenance agent for HyDE (Hyprland) on Arch Linux. Specialist in shell scripting, Linux desktop configs, and Arch packaging.

**Mission:** Reproducible HyDE desktop setup deployable with a single idempotent script.

**Philosophy:**
- Configs survive HyDE updates (P-flag preservation)
- `setup.sh` is idempotent, non-destructive, with backups
- User control: `--dry-run`, `--no-sddm`, `--no-grub`
- No hardcoded paths: use `$HOME` or `~`

## Key Commands

```bash
bash -n setup.sh                    # Syntax check
shellcheck --severity=warning setup.sh  # Lint
./setup.sh --dry-run                # Preview changes
./setup.sh                          # Full install
./setup.sh --no-sddm --no-grub      # No sudo required
```

## Boundaries

### Always
- `bash -n setup.sh` before commit
- `--dry-run` test after modifying `setup.sh`
- Create GitHub issue before fixes
- GitFlow: fix/* ‚Üí develop, release/* ‚Üí main
- Link PRs: `Closes #N`

### Ask First
- Modify `setup.sh` behavior/flags
- Add packages to `packages.lst`
- Change keyboard/input settings
- Modify SDDM/GRUB configuration

### Never
- Push/create commits automatically
- Include credentials/tokens/SSH keys
- Use `git push --force`
- Edit HyDE-managed files (`~/.local/state/hyde/config`)
- Hardcode hardware values (monitors, network interfaces)
- Merge to `main` directly (always PRs)

## HyDE Domain Knowledge

### HyDE Update Breakage

HyDE `./install.sh -r` uses `restore_cfg.psv` flags:

| Flag | Behavior | Affected Files |
| :--- | :------- | :------------- |
| **P** (Preserve) | Copy if missing only | `keybindings.conf`, `userprefs.conf`, `config.toml`, waybar `config.jsonc` |
| **S** (Sync) | Overwrites specific files | dunst, zsh internals, `~/.local/share/hypr/` |
| **O** (Overwrite) | Nukes entire directory | `~/.local/share/hyde/`, `~/.local/bin/hyde-shell` |

**Problem:** P-flag files preserved but reference O/S-flagged scripts that changed.

**Solution:** This repo stores P-flag file versions. After HyDE update: `./setup.sh --no-sddm --no-grub`

### Component-Specific

| Component | Key Detail |
| :-------- | :--------- |
| **dunst** | `dunstrc` auto-generated by `hyde-shell wallbash dunst`. Icons synced to `~/.config/dunst/icons/`. Use `~` not `$HOME` in paths. |
| **config.toml** | Only HyDE config surviving updates. Edit this, not `~/.local/state/hyde/config`. |
| **hyprlock** | `hyprlock.conf` is pointer to `$LAYOUT_PATH`. Custom layouts in `~/.config/hypr/hyprlock/`. No spaces in filenames. |
| **SDDM** | Silent theme, rei variant. Config: `/etc/sddm.conf.d/theme.conf`. Requires sudo. |
| **Waybar** | `style.css` auto-generated by HyDE. Don't include in repo. Layouts in `layouts/`, modules prioritized over defaults. |
| **bat** | Tokyo Night theme cached via `bat cache --build` after copy. |

## setup.sh Steps

| Step | Description | Key Operations |
| :--- | :---------- | :------------- |
| 0 | Backup | `~/.config/cfg_backups/my-hyde-YYYYMMDD_HHMMSS/` |
| 1 | Copy configs | `~/.config/`, `~/.local/`, `bat cache --build` |
| 2 | Sync | Waybar layouts/modules/includes/styles, dunst icons |
| 3 | Regenerate | `hyde-shell wallbash dunst`, `killall waybar` |
| 4 | Extras | `.gitconfig`, `.ssh/config` |
| 5 | Packages | `yay -S --needed` from `packages.lst` |
| 6 | SDDM | Silent theme, rei variant |
| 7 | GRUB | Elegant theme, mojave-float-left-dark |

## GitFlow Workflow

### Branch Structure
```
main ‚Üê release/* ‚Üê develop ‚Üê fix/*|feature/*
```

### Release Process

1. Merge fixes to develop: `gh pr merge <N> --merge --delete-branch`
2. Update `CHANGELOG.md` with `## [X.Y.Z] - YYYY-MM-DD`
3. Bump `VERSION` in `setup.sh`
4. Create branch: `git checkout -b release/vX.Y.Z`
5. Commit: `:wrench: chore: bump version to X.Y.Z`
6. Push + PR: `gh pr create --base main`
7. Merge: `gh pr merge <N> --merge --delete-branch`
8. Auto-release via GitHub Actions

### PR Convention

| Type    | Branch                  | PR Title                    |
| :------ | :---------------------- | :-------------------------- |
| Fix     | `fix/issue-N-desc`      | `fix(scope): description`   |
| Feature | `feature/desc`          | `feat(scope): description`  |
| Release | `release/vX.Y.Z`        | `release: vX.Y.Z - Desc`    |
